Saved DCRs

Every day you generate is stored here. Filter by movie or date range, then View DCR to re-open and re-print any past day.

Ticket serial numbers

For each screen, set the starting serial for each class, effective from a date. From that date the serials run continuously through every show in date & time order. Add another row only when a roll resets or you want to restart from a new number.

Opening totals (Previous)

Starting in the middle of a release? Instead of entering every past DCR, add the running totals so far for each Screen × Movie, as of the day before you start here. The tool then continues the Previous / Total / T net figures from these numbers — just like the serial starting points.

Movies

Add each film with its distributor, release date and distributor share %. Share auto-fills daily entries (override per day if needed). Running Day on the DCR is counted from the release date.

Price cards

Each card is a complete gross-price set for one screen's classes. Pick the screen, then add cards. Taxes are split out automatically (see Screens & Tax).

Cinema details

Class catalog

Create the classes used across your cinema (name + GST%). Assign them to individual screens — with their own seat counts — in the Screens section below.

Tax & charges

E-Tax & GST are chosen by the printed ticket price (gross): above the threshold uses one rate pair, at/below uses the other.

Rep batta is charged per movie per day, by the number of shows: 1 show, 2–4 shows, or 5 or more shows.

Screens

Name each screen and tick which catalog classes it has, with seat counts. Set ticket prices for each screen in the Price Cards tab.

Data file & backup

Best option: link a data file in your project folder (button in the bar at the top) — the tool then auto-saves everything (screens, classes, price cards, movies, serial numbers, settings and daily entries) there on every change. The buttons below make a one-off copy you can move or keep as a snapshot.

Saved days

Bulk upload (CSV)

Add many past days at once. Download the template, fill one row per show (rows sharing the same Date + Movie + Screen become one day), then upload. You'll see a preview and any errors before anything saves — importing only adds/updates those days, it never wipes other data.

Daily F&B (DSR upload)

Upload the day's DSR_Items_Breakup CSV exported from your POS. The date is read from the CSV header — one upload = one day. If a day already exists, the new upload replaces it.

Bulk upload — many DSR files

Got DSRs for several days? Select all the CSVs at once and import them in one go. Existing entries on those dates will be replaced.

Bulk upload (single combined CSV)

For backfills or corrections: one CSV file with many days. Each row = one item (Date, ItemName, Qty are the only required columns). Existing entries for those dates are replaced on import.
